## 📋 概述

全新设计的浏览器管理界面，**内置分组功能**，采用现代化卡片式布局，操作更直观、更高效。

### 核心改进

1. **✅ 分组功能内置** - 不再是单独的页面，而是集成在左侧树形结构中
2. **✅ 卡片式展示** - 现代化、美观、信息一目了然
3. **✅ 双视图切换** - 支持卡片视图和列表视图
4. **✅ 智能搜索** - 实时搜索浏览器名称、UA、分组
5. **✅ 批量操作** - 选中多个浏览器后显示批量操作栏
6. **✅ 右键菜单** - 减少按钮数量，操作更便捷
7. **✅ 统计面板** - 总数、运行中、分组数一目了然

## 🚀 快速开始

### 1. 打开浏览器管理

在主菜单点击 **"浏览器管理"**，系统会自动加载新版界面。

### 2. 创建分组

1. 在左侧分组树下方点击 **"➕ 新建分组"**
2. 输入分组名称（如："工作浏览器"）
3. 输入描述（可选）
4. 点击 **"确定"**

### 3. 创建浏览器

1. 点击右上角 **"➕ 新建浏览器"** 按钮
2. 填写浏览器信息
3. 选择指纹配置
4. 选择所属分组
5. 点击 **"创建"**

### 4. 启动浏览器

**方式 1：卡片视图**
- 点击浏览器卡片上的 **"▶️"** 按钮

**方式 2：列表视图**
- 右键点击浏览器 → 选择 **"▶️ 启动"**

**方式 3：批量启动**
- 勾选多个浏览器
- 点击批量操作栏的 **"▶️ 批量启动"**

## 🎯 核心功能

### 1. 分组管理

#### 创建分组
```
左侧分组树 → [➕ 新建分组] → 输入信息 → 确定
```

#### 编辑分组
```
选中分组 → [✏️ 编辑分组] → 修改信息 → 确定
```

#### 删除分组
```
选中分组 → [🗑️ 删除分组] → 确认
注意：分组下的浏览器会变为"未分组"
```

#### 移动浏览器到分组
```
方式 1：右键浏览器 → 📁 移动到分组 → 选择目标分组
方式 2：批量选中 → [📁 批量移动] → 选择目标分组
```

### 2. 浏览器操作

#### 单个操作

**卡片视图**:
- **▶️** - 启动浏览器
- **✏️** - 编辑浏览器
- **⋮** - 更多操作（更换指纹、移动分组、清除会话、删除）

**列表视图**:
- 右键点击浏览器 → 选择操作

#### 批量操作

1. **勾选多个浏览器**（卡片视图右上角的复选框）
2. **批量操作栏自动显示**
3. **选择操作**：
   - **▶️ 批量启动** - 启动所有选中的浏览器
   - **🔄 批量更换指纹** - 为所有选中的浏览器更换指纹
   - **📁 批量移动** - 将所有选中的浏览器移动到指定分组
   - **🗑️ 批量删除** - 删除所有选中的浏览器

### 3. 搜索和筛选

#### 实时搜索
```
在顶部搜索框输入关键词 → 自动过滤显示
支持搜索：浏览器名称、User-Agent、分组名称
```

#### 分组筛选
```
点击左侧分组树中的分组 → 右侧只显示该分组的浏览器
```

#### 排序
```
右上角排序下拉框 → 选择排序方式：
- 按名称排序
- 按创建时间
- 按启动次数
```

### 4. 视图切换

#### 卡片视图（默认）
- 美观、直观
- 信息丰富
- 适合浏览和快速操作

#### 列表视图
- 紧凑、高效
- 显示更多列
- 适合批量管理

**切换方式**:
```
统计面板右侧 → [📇 卡片] / [📋 列表]
```

## 🎯 使用场景

### 场景 1：按项目管理浏览器

```
1. 创建分组："客户A"、"客户B"、"内部测试"
2. 为每个项目创建专用浏览器
3. 点击分组 → 只显示该项目的浏览器
4. 批量启动该项目的所有浏览器
```

### 场景 2：批量更新指纹

```
1. 生成新的指纹配置
2. 勾选所有需要更新的浏览器
3. 点击 [🔄 批量更换指纹]
4. 选择新指纹 → 确认
5. 所有浏览器下次启动时使用新指纹
```

### 场景 3：快速查找浏览器

```
1. 搜索框输入关键词（如："测试"）
2. 系统自动过滤显示
3. 找到目标浏览器 → 点击 ▶️ 启动
```

### 场景 4：清理测试环境

```
1. 点击"测试"分组
2. 勾选所有测试浏览器
3. 点击 [🗑️ 批量删除]
4. 确认 → 完成清理
```

## ⚠️ 注意事项

### 1. 分组删除

- 删除分组时，分组下的浏览器**不会被删除**
- 浏览器会自动移动到"未分组"

### 2. 批量操作

- 批量启动可能消耗大量资源
- 建议一次启动不超过 5 个浏览器

### 3. 搜索功能

- 搜索是实时的，输入即过滤
- 清空搜索框可恢复完整列表

### 4. 视图切换

- 切换视图不会影响数据
- 选中状态会保留
